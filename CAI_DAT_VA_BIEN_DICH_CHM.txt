1. FILE CHM LÀ GÌ?
================================================================================

CHM (Compiled HTML Help) là:
• Định dạng file trợ giúp của Microsoft
• Tích hợp nhiều file HTML thành 1 file duy nhất
• Có mục lục, tìm kiếm, index tự động
• Dễ chia sẻ và phân phối
• Chạy trực tiếp trên Windows (không cần cài đặt thêm)

Ưu điểm:
✅ Nhỏ gọn (1 file duy nhất)
✅ Có chức năng tìm kiếm mạnh mẽ
✅ Giao diện chuyên nghiệp
✅ Dễ sử dụng
✅ Không cần Internet

Nhược điểm:
❌ Chỉ chạy tốt trên Windows
❌ Cần công cụ đặc biệt để tạo file

3. CÀI ĐẶT HTML HELP WORKSHOP
================================================================================

🔽 BƯỚC 1: DOWNLOAD
--------------------
• Truy cập: https://www.microsoft.com/en-us/download/details.aspx?id=21138
• Hoặc tìm "HTML Help Workshop download" trên Google
• Click nút "Download"
• Lưu file: htmlhelp.exe (khoảng 4MB)

🔧 BƯỚC 2: CÀI ĐẶT
--------------------
1. Double-click file htmlhelp.exe
2. Click "Yes" nếu có cảnh báo User Account Control
3. Chọn ngôn ngữ: English (khuyến nghị)
4. Click "Next"
5. Accept license agreement
6. Click "Next"
7. Chọn thư mục cài đặt (mặc định: C:\Program Files (x86)\HTML Help Workshop\)
8. Click "Next"
9. Click "Install"
10. Đợi cài đặt hoàn tất
11. Click "Finish"

✅ BƯỚC 3: KIỂM TRA CÀI ĐẶT
-----------------------------
• Mở Start Menu
• Tìm "HTML Help Workshop"
• Nếu thấy chương trình → CÀI ĐẶT THÀNH CÔNG ✓

Hoặc kiểm tra file:
• Mở: C:\Program Files (x86)\HTML Help Workshop\
• Phải có file: hhc.exe

4. BIÊN DỊCH FILE CHM
================================================================================

📌 PHƯƠNG PHÁP 1: SỬ DỤNG BUILD.BAT (ĐƠN GIẢN NHẤT)
----------------------------------------------------

Bước 1: Mở thư mục help_docs
Bước 2: Double-click file: build.bat
Bước 3: Xem kết quả:

   • Nếu thành công:
     ✅ Hiện "Build complete!"
     ✅ File QuanLyDocGia_GiaHanThe.chm được tạo
     ✅ Nhấn phím bất kỳ để đóng

   • Nếu lỗi:
     ❌ Hiện "HTML Help Workshop not found!"
     ❌ Kiểm tra lại đã cài đặt chưa
     ❌ Hoặc sửa đường dẫn trong build.bat

📌 PHƯƠNG PHÁP 2: SỬ DỤNG HTML HELP WORKSHOP
--------------------------------------------

Bước 1: Mở HTML Help Workshop
   • Start Menu > HTML Help Workshop
   • Hoặc chạy: C:\Program Files (x86)\HTML Help Workshop\hhw.exe

Bước 2: Mở project
   • File > Open...
   • Chọn file: help_project.hhp (trong thư mục help_docs)
   • Click "Open"

Bước 3: Xem trước project (tùy chọn)
   • Xem tab [Project] - danh sách file
   • Xem tab [Contents] - mục lục

Bước 4: Biên dịch
   • Click nút "Compile" trên toolbar
   • Hoặc: File > Compile...
   • Hoặc nhấn: F9
   • Cửa sổ "Create a compiled file" hiện ra
   • Click "Compile"

Bước 5: Đợi biên dịch
   • Xem log trong cửa sổ
   • Nếu thành công: "Compile time: X seconds"
   • Nếu có lỗi: Đọc thông báo lỗi

Bước 6: Xem kết quả
   • Click "View compiled file" (nếu thành công)
   • Hoặc mở thư mục help_docs, tìm file .chm

📌 PHƯƠNG PHÁP 3: SỬ DỤNG COMMAND LINE
---------------------------------------

Bước 1: Mở Command Prompt
   • Nhấn Win+R
   • Gõ: cmd
   • Enter

Bước 2: Di chuyển đến thư mục
   • Gõ lệnh: cd "C:\CNPM\New folder\ute-library-management-new\help_docs"
   • (Thay đường dẫn cho đúng với máy bạn)
   • Enter

Bước 3: Chạy lệnh biên dịch
   • Gõ lệnh:
     "C:\Program Files (x86)\HTML Help Workshop\hhc.exe" help_project.hhp
   • Enter

Bước 4: Xem kết quả
   • Nếu thành công: File .chm được tạo
   • Nếu lỗi: Đọc thông báo lỗi

5. SỬ DỤNG FILE CHM
================================================================================

🔓 MỞ FILE CHM
--------------

Cách 1: Double-click
   • Double-click file: QuanLyDocGia_GiaHanThe.chm
   • File sẽ mở trong CHM Viewer

Cách 2: Chuột phải
   • Chuột phải vào file .chm
   • Chọn "Open"

⚠️ NẾU BỊ CHẶN (Windows Security)
----------------------------------

Nếu file CHM mở nhưng nội dung trống hoặc bị chặn:

Bước 1: Đóng file CHM
Bước 2: Chuột phải vào file .chm
Bước 3: Chọn "Properties"
Bước 4: Ở tab "General", phía dưới:
   • Tìm dòng: "Security: This file came from another computer..."
   • Tích vào ô: "Unblock"
Bước 5: Click "Apply"
Bước 6: Click "OK"
Bước 7: Mở lại file CHM

✅ SỬ DỤNG FILE CHM
-------------------

Giao diện CHM gồm 3 phần:

1. Bên trái - Navigation:
   • Tab "Contents": Mục lục (Table of Contents)
   • Tab "Index": Chỉ mục từ khóa
   • Tab "Search": Tìm kiếm toàn văn

2. Bên phải - Content:
   • Hiển thị nội dung trang

3. Toolbar:
   • Hide/Show: Ẩn/hiện panel trái
   • Back/Forward: Quay lại/Tiến tới
   • Print: In trang hiện tại

💡 MẸO SỬ DỤNG:
----------------
• Click vào mục lục để điều hướng
• Sử dụng Search để tìm nhanh
• Ctrl+F để tìm trong trang hiện tại
• Có thể bookmark các trang (Favorites)
• In ra giấy nếu cần: File > Print

6. XỬ LÝ SỰ CỐ
================================================================================

❌ SỰ CỐ 1: "hhc.exe is not recognized"
----------------------------------------
Nguyên nhân: Chưa cài HTML Help Workshop hoặc đường dẫn sai
Giải pháp:
   1. Kiểm tra đã cài đặt chưa
   2. Kiểm tra đường dẫn: C:\Program Files (x86)\HTML Help Workshop\hhc.exe
   3. Sửa đường dẫn trong build.bat nếu cài ở nơi khác

❌ SỰ CỐ 2: "Error: Cannot open file..."
-----------------------------------------
Nguyên nhân: Thiếu file hoặc đường dẫn sai
Giải pháp:
   1. Kiểm tra tất cả file HTML đều có trong thư mục
   2. Kiểm tra tên file trong help_project.hhp
   3. Đảm bảo chạy lệnh trong thư mục help_docs

❌ SỰ CỐ 3: File CHM mở nhưng trống
------------------------------------
Nguyên nhân: Windows chặn file từ nguồn không tin cậy
Giải pháp:
   1. Properties > Unblock (xem hướng dẫn trên)
   2. Copy file CHM vào thư mục khác thử
   3. Kiểm tra file style.css có cùng thư mục khi biên dịch

❌ SỰ CỐ 4: Tiếng Việt hiển thị lỗi
------------------------------------
Nguyên nhân: Encoding không đúng
Giải pháp:
   1. Mở lại file HTML bằng Notepad++
   2. Encoding > Convert to UTF-8
   3. Save
   4. Biên dịch lại

❌ SỰ CỐ 5: CSS không hiển thị
-------------------------------
Nguyên nhân: Thiếu file style.css hoặc đường dẫn sai
Giải pháp:
   1. Kiểm tra file style.css có trong thư mục
   2. Kiểm tra đã thêm vào [FILES] trong help_project.hhp
   3. Biên dịch lại

❌ SỰ CỐ 6: Mục lục không hiển thị
-----------------------------------
Nguyên nhân: File help_toc.hhc bị lỗi hoặc thiếu
Giải pháp:
   1. Kiểm tra file help_toc.hhc có đúng format XML
   2. Kiểm tra đường dẫn trong help_project.hhp
   3. Mở bằng HTML Help Workshop để kiểm tra

❌ SỰ CỐ 7: "Warning: HHC5003"
-------------------------------
Nguyên nhân: File HTML có lỗi cú pháp (thường không nghiêm trọng)
Giải pháp:
   1. Nếu file CHM vẫn tạo được → Bỏ qua warning
   2. Nếu muốn sửa: Mở file HTML báo lỗi, kiểm tra HTML syntax
